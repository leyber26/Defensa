Por qué evolucionar.

Objetivos del Sistema

Objetivo General

Desarrollar un sistema de información web para el cálculo y gestión de indicadores de rendimiento bajo la norma ISO 11620 en la Red de Bibliotecas de la Universidad de Carabobo.

1. Diagnosticar

Examinar los procesos actuales de recolección y cálculo de datos estadísticos bibliotecarios.

2. Analizar

Determinar los requerimientos funcionales y no funcionales que debe satisfacer el software (Ingeniería de Requisitos).

3. Diseñar

Modelar la arquitectura de la base de datos relacional y construir la interfaz gráfica de usuario moderna.

4. Desarrollar

Codificar la solución tecnológica aplicando los lenguajes, stacks y frameworks previamente seleccionados.

5. Evaluar

Efectuar pruebas exhaustivas de funcionalidad, caja blanca e integración para la validación con los usuarios finales de la UC.

Arquitectura de Datos

Modelo Entidad-Relación

Estructura normalizada en PostgreSQL que garantiza la integridad referencial para auditorías eficientes.

SQL Schema
CREATE TABLE bibliotecas (
    -- Entidad Principal de la Red Institucional
    id_biblioteca   SERIAL PRIMARY KEY,
    nombre          VARCHAR(100) NOT NULL,
    facultad        VARCHAR(100) NOT NULL
);

CREATE TABLE usuarios (
    -- Control de Acceso RBAC (Roles Autorizados)
    id_usuario      SERIAL PRIMARY KEY,
    id_biblioteca   INTEGER REFERENCES bibliotecas(id_biblioteca),
    rol             VARCHAR(20) NOT NULL -- 'Admin_Regional' / 'Analista'
);

CREATE TABLE registros_datos (
    -- Data en crudo importada a la plataforma
    id_registro     SERIAL PRIMARY KEY,
    id_biblioteca   INTEGER REFERENCES bibliotecas(id_biblioteca),
    periodo         DATE NOT NULL,
    valor_metricas  JSONB NOT NULL
);

CREATE TABLE indicadores_calculados (
    -- Resultados Algorítmicos ISO 11620 con Foreign Keys estrictas
    id_resultado    SERIAL PRIMARY KEY,
    id_registro     INTEGER REFERENCES registros_datos(id_registro),
    puntuacion_b1   DECIMAL(5,2),
    puntuacion_b2   DECIMAL(5,2)
);

Ingeniería Web

Arquitectura del Stack

HTML5

Estructura Semántica. Base del markup moderno. El sistema presenta formularios estructurados y accesibles por la norma.

JavaScript

Lógica Frontend. El motor de interactividad. Maneja respuestas dinámicas, validaciones en tiempo real y gráficas Chart.js.

PHP 7.4

Motor Lógico. Procesa los algoritmos ISO 11620 del backend y gestiona sesiones de usuario de forma segura con patrón MVC.

PostgreSQL

Persistencia. RDBMS para mantener integridad referencial y proteger los datos históricos de toda la Red de Bibliotecas.

Chart.js

Visualización. Genera gráficos de barras, líneas y polar con datos ISO en tiempo real del dashboard institucional.

html2pdf.js

Reportes PDF. Exporta informes ISO oficiales directamente desde el navegador sin dependencias de servidor adicionales.

Tailwind CSS

UI Moderna. Estilos utility-first con glassmorphism, animaciones y responsive design a lo largo de toda la plataforma.

Git

Versión y Colaboración. Control de cambios completo con GitHub para trazabilidad del desarrollo a lo largo del proyecto.
